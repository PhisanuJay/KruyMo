## ระบบเว็บไซต์สำหรับเช่าชุดครุยออนไลน์ (KruyMo)

## 1. บทคัดย่อ (Abstract)

โครงงานนี้มีวัตถุประสงค์เพื่อพัฒนา **"KruyMo (ครุยโม้)"** ซึ่งเป็นระบบเว็บไซต์สำหรับให้บริการเช่าชุดครุยออนไลน์แบบครบวงจร เพื่อแก้ไขปัญหาหลักที่พบในกระบวนการเช่าชุดครุยรูปแบบดั้งเดิม ซึ่งผู้ใช้บริการมักจะประสบความไม่สะดวกจากการที่ต้องเดินทางมาติดต่อที่หน้าร้านหลายครั้งเพื่อเลือกชุด ตรวจสอบไซส์ จองคิว หรือติดตามสถานะ

อีกทั้งฝั่งผู้ให้บริการ (ร้านเช่า) ยังคงใช้ระบบจดบันทึกข้อมูลแบบแมนนวล (Manual) ซึ่งนำไปสู่ความผิดพลาดในการจัดการสต็อกสินค้าและการบันทึกคิวการจอง

คณะผู้พัฒนาจึงได้นำเทคโนโลยีเว็บแอปพลิเคชันมาประยุกต์ใช้ โดยทำการออกแบบระบบด้วย Figma และพัฒนาผ่านเครื่องมือ Visual Studio Code ร่วมกับระบบจัดการฐานข้อมูล Local Storage

ระบบประกอบด้วย 4 โมดูลการทำงานหลัก ได้แก่

1. ระบบลูกค้า
2. ระบบพนักงาน
3. ระบบผู้ดูแลระบบ
4. ระบบจัดการธุรกรรมชำระเงิน

ผลลัพธ์ที่คาดหวังหลังการติดตั้งระบบคือจะสามารถลดขั้นตอนและค่าใช้จ่ายในการเดินทางของลูกค้าลงอย่างมีนัยสำคัญ ลดอัตราความผิดพลาดด้านข้อมูลของร้านค้า และเพิ่มขีดความสามารถในการบริหารจัดการคิวและคลังสินค้าได้อย่างเป็นระบบตามหลักวิศวกรรมซอฟต์แวร์

## 2. บทนำ (Introduction)

### 2.1 ความเป็นมาและความสำคัญของโครงงาน

การเข้ารับพระราชทานปริญญาบัตรถือเป็นช่วงเวลาและเหตุการณ์ครั้งสำคัญที่สุดเหตุการณ์หนึ่งในชีวิตของนักศึกษาระดับอุดมศึกษา อย่างไรก็ตาม กระบวนการในการเข้าถึงการเช่าชุดครุยในปัจจุบันยังมีข้อจำกัดและสร้างความไม่สะดวกให้แก่ผู้ใช้บริการหลายประการ

ปัญหาสำคัญคือ ลูกค้าจำเป็นต้องเดินทางมาที่หน้าร้านค้าหลายระลอก ตั้งแต่ขั้นตอนการเลือกดูประเภทชุด การวัดขนาดและเลือกไซส์เสื้อผ้า การติดต่อรับชุดครุย ตลอดจนการส่งคืนชุดเมื่อสิ้นสุดพิธี

ในขณะเดียวกัน ฝั่งผู้ประกอบการหรือร้านค้าเช่าชุดส่วนใหญ่ยังขาดระบบซอฟต์แวร์เฉพาะทาง โดยส่วนใหญ่ยังคงพึ่งพาวิธีการจดบันทึกข้อมูลลงกระดาษหรือสมุดบัญชีด้วยลายมือ ซึ่งก่อให้เกิดความเสี่ยงสูงต่อการทำข้อมูลสูญหาย ปัญหาการบันทึกรายการจองซ้ำซ้อน (Overbooking) หรือความคลาดเคลื่อนในการนับจำนวนสต็อกชุดคงเหลือ

นอกจากนี้ การที่ระบบเดิมไม่มีช่องทางให้ลูกค้าติดตามสถานะคำขอเช่าได้ด้วยตนเอง ส่งผลให้เกิดภาระงานที่พนักงานต้องคอยตอบคำถามผ่านโทรศัพท์อยู่ตลอดเวลา

ด้วยเหตุนี้ คณะผู้จัดทำจึงเล็งเห็นโอกาสในการนำเทคโนโลยีสารสนเทศเข้ามาแก้ไขปัญหา โดยการพัฒนาเว็บแอปพลิเคชันภายใต้ชื่อโครงงาน **"KruyMo (ครุยโม้)"** ขึ้นมา เพื่อย้ายกระบวนการดำเนินงานทั้งหมดไปอยู่บนแพลตฟอร์มออนไลน์ ซึ่งจะช่วยยกระดับการให้บริการและเพิ่มประสิทธิภาพการบริหารจัดการเชิงธุรกิจอย่างยั่งยืน

### 2.2 วัตถุประสงค์

1. เพื่อวิเคราะห์ ออกแบบ และพัฒนาระบบเว็บไซต์สำหรับเช่าชุดครุยออนไลน์ที่มีเสถียรภาพและใช้งานได้จริง
2. เพื่อเพิ่มช่องทางและความสะดวกให้ลูกค้าสามารถดำเนินกิจกรรมเลือกชุด ระบุไซส์ และจองคิวเช่าผ่านเว็บไซต์ได้จากทุกสถานที่
3. เพื่อลดจำนวนครั้งและลดความจำเป็นในการเดินทางมาบริหารจัดการธุรกรรมที่หน้าร้านของลูกค้า
4. เพื่อลดข้อผิดพลาดในการบันทึกและจัดการข้อมูลคิวการจองอันเกิดจากการทำงานในระบบแมนนวล (Manual Error)
5. เพื่อส่งเสริมและสนับสนุนให้ทางร้านค้าสามารถบริหารจัดการคลังสินค้า สถิติการเช่า และรายได้รวมได้อย่างมีประสิทธิภาพสูงสุด

### 2.3 ขอบเขตของระบบ (System Scope)

ระบบแบ่งขอบเขตและหน้าที่การทำงานออกตามสิทธิ์ของผู้ใช้งาน (User Roles) 3 กลุ่มหลัก ดังนี้

#### 2.3.1 ขอบเขตด้านผู้ใช้งาน (User Roles)

- **Customer (ลูกค้า):** ทำการสมัครสมาชิก, เข้าสู่ระบบ, เข้าชมคู่มือแนะนำการเช่าชุด, ค้นหาชุดครุยตามเงื่อนไขมหาวิทยาลัยหรือคณะวิชา, ตรวจสอบรายละเอียดและขนาดไซส์, ดำเนินการจองชุดพร้อมระบุวันเวลาเช่า, ชำระเงินผ่านระบบและอัปโหลดสลิป, รับแจ้งเตือนยืนยัน, ส่งคำขอยกเลิกการจอง และเรียกดูประวัติธุรกรรมย้อนหลังได้
- **Staff (พนักงานร้าน):** เรียกดูข้อมูลคิวการจองล่วงหน้าเพื่อเตรียมงาน, ตรวจสอบความพร้อมของตัวชุดครุย, ดำเนินการอนุมัติการจองหลังจากตรวจสลิปโอนเงิน, อัปเดตสถานะการส่งมอบและการรับคืนชุด, บันทึกผลการตรวจสอบสภาพชุดภายหลังการใช้งาน, ตรวจสอบกรณีลูกค้ายกเลิก และบันทึกข้อมูลธุรกรรมหักค่าปรับ/คืนเงินมัดจำ
- **Admin (ผู้ดูแลระบบ):** บริหารจัดการโครงสร้างข้อมูลพื้นฐานของระบบเช่าทั้งหมด (เพิ่ม, เรียกดู, แก้ไข, ลบข้อมูลชุดครุย ข้อมูลมหาวิทยาลัย และคณะวิชา) และเข้าถึง Dashboard เพื่อประเมินสถิติเชิงบริหาร

#### 2.3.2 ขอบเขตด้านฟังก์ชัน (Functional Features)

- **ระบบการจองและคำนวณไซส์:** ตรวจสอบสิทธิ์การจองแบบเรียลไทม์เพื่อไม่ให้เกิดการจองซ้ำในไซส์เดียวกันและเวลาเดียวกัน
- **ระบบธุรกรรมการเงินและมัดจำ:** บริหารสถานะธุรกรรมทางการเงิน (เงินโอน, QR Code, PromptPay และเงินสดหน้าร้าน) พร้อมฟังก์ชันหักยอดเงินมัดจำกรณีส่งคืนชุดล่าช้าหรือชำรุด
- **ระบบแสดงผลสารสนเทศ (Dashboard):** ประมวลผลข้อมูลรายได้รวมยอดขาย, อัตราสถิติความนิยมในการเช่าจำแนกตามสถาบัน และยอดคงคลังคงเหลือ

## 3. การศึกษาที่เกี่ยวข้อง (Literature Review)

คณะผู้จัดทำได้ทำการรวบรวมศึกษาทฤษฎี เอกสารวิจัย และระบบงานใกล้เคียงเพื่อนำมาประยุกต์ใช้ในการพัฒนาโครงการ ดังนี้

- **การศึกษากระบวนการ E-Commerce และการจองคิวบริการออนไลน์:** ศึกษาแบบแผนพฤติกรรมผู้บริโภคในการเลือกซื้อและเช่าสินค้าออนไลน์เพื่อออกแบบ Workflow การกรอกข้อมูลและการอัปโหลดสลิปธนาคารให้สั้น กระชับ และเข้าใจง่ายที่สุด
- **การศึกษาเครื่องมือทางเทคโนโลยี:**
  - **Figma:** ใช้ในการทำแบบจำลองหน้าจอความละเอียดสูง (High-Fidelity Prototype) เพื่อทดสอบ User Experience ก่อนการเขียนโค้ด
  - **Visual Studio Code:** ซอฟต์แวร์สภาพแวดล้อมการพัฒนาหลัก (IDE) รองรับการเขียน Code และสคริปต์การทำงาน
  - **Git และ GitHub:** นำระบบคลังข้อมูลออนไลน์มาบริหารการพัฒนาโปรเจกต์ ร่วมกันทำงานในทีม และควบคุมเวอร์ชันของระบบ
  - **Local Storage:** ใช้สำหรับจัดเก็บข้อมูลภายในเว็บเบราว์เซอร์ของผู้ใช้งาน เช่น ข้อมูลการใช้งานหรือข้อมูลที่จำเป็นสำหรับการทำงานของระบบ โดยสามารถเรียกใช้งานข้อมูลได้อย่างรวดเร็วโดยไม่ต้องพึ่งพาฐานข้อมูลภายนอก

## 4. การวิเคราะห์ระบบ (System Analysis)

### 4.1 Problem Statement (ปัญหาของระบบเดิม)

ระบบเดิมใช้การประสานงานแบบแมนนวลและจดบันทึกบนกระดาษเป็นหลัก ส่งผลให้ค้นหาข้อมูลอ้างอิงออเดอร์เก่าได้ยาก ข้อมูลสต็อกสินค้าไม่มีการอัปเดตแบบทันที ทำให้หน้าร้านประสบปัญหาปล่อยเช่าชุดเกินจำนวนสต็อกที่มีอยู่จริง (Inventory Over-allocation) สร้างความเสียหายและส่งผลกระทบต่อภาพลักษณ์ของร้านค้าในวันงานสำคัญของลูกค้า

### 4.2 Requirement Analysis

#### Functional Requirements

- ระบบต้องจัดทำหน้าลงทะเบียนและระบบการลงชื่อเข้าใช้งานที่จำแนกสิทธิ์ผู้ใช้ได้อย่างถูกต้อง
- ระบบต้องมีตัวกรองสำหรับค้นหาชุดครุยแยกเป็นระดับขั้นปริญญา มหาวิทยาลัย และคณะวิชาได้โดยไม่สับสน
- ระบบต้องรองรับการบันทึกรูปภาพสลิปธุรกรรมและส่งอีเมลยืนยันการจองหาผู้ใช้ได้ทันทีหลังพนักงานกดอนุมัติ

#### Non-functional Requirements

- **Usability:** ส่วนติดต่อผู้ใช้งาน (UI) ต้องได้รับการออกแบบให้สอดคล้องกับหลัก Responsive Design เพื่อรองรับสมาร์ตโฟนและแท็บเล็ต
- **Performance:** อัตราการประมวลผลและการสืบค้นสต็อกสินค้าคงคลังต้องใช้เวลาโหลดไม่เกิน 3 วินาทีต่อหนึ่งการค้นหา
- **Security:** รหัสผ่านของผู้ใช้งานทั้งหมดต้องได้รับการเข้ารหัส (Encryption) ก่อนจัดเก็บลงฐานข้อมูล

## 7. ผลที่คาดว่าจะได้รับ (Expected Benefits)

- ผู้ใช้บริการได้รับความพึงพอใจเพิ่มขึ้นจากฟังก์ชันการจองชุดครุยแบบออนไลน์ล่วงหน้าได้ตลอด 24 ชั่วโมง
- ช่วยลดปริมาณงานและระยะเวลาในการประสานงานของร้านค้าลง
- เพิ่มความแม่นยำในการบันทึกรายได้และควบคุมจำนวนสต็อกชุดครุย ไม่ให้เกิดข้อผิดพลาดในการดำเนินงาน
- ส่งเสริมการโปร่งใส ตรวจสอบสถานะการเงินและการคืนเงินมัดจำผ่านระบบอิเล็กทรอนิกส์
