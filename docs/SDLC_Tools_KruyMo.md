# การประยุกต์ใช้เครื่องมือในกระบวนการ SDLC

โครงงาน **KruyMo — ระบบเช่าชุดครุยบัณฑิตออนไลน์** ได้ประยุกต์ใช้เครื่องมือต่างๆ ตามขั้นตอนของ Software Development Life Cycle (SDLC) เพื่อให้การวางแผน การออกแบบ การพัฒนา การทดสอบ และการเผยแพร่ระบบเป็นไปอย่างเป็นระบบ ดังนี้

---

## 1. Planning (การวางแผน)

### เครื่องมือที่ใช้
- Trello
- Google Docs
- Google Meet

### เหตุผลที่เลือกใช้
ใช้สำหรับรวบรวมความต้องการของผู้ใช้งาน (Requirement) วางแผนการดำเนินโครงการ แบ่งหน้าที่ของสมาชิกในทีม และประชุมติดตามความคืบหน้า ทำให้การทำงานเป็นระบบและสามารถตรวจสอบงานได้ง่าย

### ตัวอย่างการใช้งานในโครงงาน
- จัดทำรายการฟังก์ชันของระบบ เช่น สมัครสมาชิกและยืนยันอีเมลด้วย OTP ค้นหาชุดครุย จองชุด ชำระเงินด้วย PromptPay ติดตามสถานะ จัดส่งด้วยแมสเซนเจอร์ รับคืนชุด และคืนเงินมัดจำ
- กำหนดระยะเวลาการพัฒนาแต่ละส่วนของระบบ ตั้งแต่โมดูลลูกค้า โมดูลพนักงาน โมดูลแอดมิน จนถึงเอกสาร UML และ UAT
- แบ่งหน้าที่ระหว่างผู้พัฒนา Frontend (หน้าลูกค้า) Backend (API และข้อมูล JSON) ระบบ Staff–Admin และงานเอกสาร/ทดสอบ

---

## 2. Analysis & Design (การวิเคราะห์และออกแบบ)

### เครื่องมือที่ใช้
- Draw.io
- Figma

### เหตุผลที่เลือกใช้
ใช้วิเคราะห์และออกแบบโครงสร้างของระบบก่อนเริ่มพัฒนา ช่วยให้ทีมเข้าใจการทำงานของระบบตรงกัน ลดข้อผิดพลาดระหว่างการพัฒนา และสามารถอ้างอิงแบบจำลองระหว่างการเขียนโค้ดได้

### ตัวอย่างการใช้งานในโครงงาน
- ออกแบบ **Use Case Diagram** เพื่อแสดงบทบาทผู้ใช้ (Customer, Staff, Admin) และฟังก์ชันหลักของระบบ
- ออกแบบ **Class Diagram** เพื่อแสดงโครงสร้างข้อมูลหลัก เช่น ผู้ใช้ การจอง การชำระเงิน ชุดครุย และสต็อก
- ออกแบบ **Sequence Diagram** เพื่อแสดงลำดับการทำงานตั้งแต่จองชุด ชำระเงิน จัดส่ง จนถึงคืนมัดจำ
- ออกแบบ **System Architecture Diagram** เพื่อแสดงชั้น Presentation, Application และ Data ของระบบ
- ออกแบบ **Wireframe / Prototype** ของเว็บไซต์ KruyMo ด้วย Figma ก่อนนำไปพัฒนาเป็นหน้าจอจริงด้วย React

> หมายเหตุ: ไฟล์แผนภาพ UML อยู่ในโฟลเดอร์ `docs/diagrams/`

## 3. Development (การพัฒนา)

### เครื่องมือที่ใช้
- Visual Studio Code
- Git
- GitHub
- React + Vite (Frontend)
- Node.js + Express (Backend)
- JSON File Storage (ชั้นข้อมูล)
- JWT / bcrypt / Nodemailer / Multer (ความปลอดภัย การยืนยันอีเมล และการอัปโหลดไฟล์)

### เหตุผลที่เลือกใช้
ใช้พัฒนาเว็บแอปพลิเคชันทั้งส่วน Frontend และ Backend พร้อมจัดเก็บข้อมูลด้วยไฟล์ JSON ซึ่งเหมาะสมกับขอบเขตของโครงงานรายวิชา และใช้ Git/GitHub ควบคุมเวอร์ชันของโค้ดเพื่อให้สมาชิกสามารถทำงานร่วมกันได้

### ตัวอย่างการใช้งานในโครงงาน
- พัฒนาระบบสมัครสมาชิก เข้าสู่ระบบ ยืนยันอีเมล และรีเซ็ตรหัสผ่านด้วย OTP
- พัฒนาระบบค้นหา/เลือกชุดครุยตามคณะ ไซส์ และระดับปริญญาตรี/โท/เอก
- พัฒนาระบบจองชุดครุย ตะกร้าสินค้า และคำนวณค่าเช่าพร้อมเงินมัดจำ
- พัฒนาระบบชำระเงิน PromptPay พร้อมอัปโหลดสลิป
- พัฒนาคิวปฏิบัติการของพนักงาน ได้แก่ ตรวจสลิป จัดเตรียมชุด มอบแมสเซนเจอร์ รับคืนชุด และคืนมัดจำ
- พัฒนา Dashboard และรายงานสำหรับผู้ดูแลระบบ รวมถึงการจัดการสต็อกและข้อมูลหลัก

## 4. Testing (การทดสอบ)

### เครื่องมือที่ใช้
- Manual Testing
- User Acceptance Testing (UAT)
- Postman (ทดสอบ API ตามความเหมาะสม)

### เหตุผลที่เลือกใช้
ใช้ตรวจสอบการทำงานของระบบและ API เพื่อให้แน่ใจว่าฟังก์ชันต่างๆ ทำงานถูกต้องตามความต้องการของผู้ใช้งานทั้ง 3 บทบาท และลดข้อผิดพลาดก่อนนำระบบไปสาธิตหรือใช้งานจริง

### ตัวอย่างการใช้งานในโครงงาน
- ทดสอบการสมัครสมาชิก การยืนยันอีเมล และการเข้าสู่ระบบ
- ทดสอบการจองชุด การชำระเงิน และการยกเลิกการจองที่ยังไม่ชำระ
- ทดสอบการตรวจสลิป การจัดส่ง การรับคืน และการคืนเงินมัดจำ
- ทดสอบ Dashboard รายงาน และการจัดการสต็อกฝั่งแอดมิน
- ทดสอบกรณีผิดปกติ เช่น รหัสผ่านผิด การเข้าหน้าผิดบทบาท และการจองช่วงวันที่สต็อกเต็ม

> เอกสารอ้างอิง: `docs/User_Acceptance_Testing.pdf` และ `docs/UAT_KruyMo_Workshop7.docx`

## 5. Deployment (การนำระบบขึ้นใช้งาน / การเผยแพร่)

### เครื่องมือที่ใช้
- GitHub
- สภาพแวดล้อมพัฒนาบนเครื่อง (localhost)

### เหตุผลที่เลือกใช้
ใช้ GitHub สำหรับจัดเก็บ Source Code และเอกสารของโครงงาน เพื่อให้สมาชิกในทีมเข้าถึง ปรับปรุง และติดตามประวัติการพัฒนาได้สะดวก ส่วนการสาธิตระบบใช้การรัน Frontend และ Backend บนเครื่องพัฒนา ซึ่งเหมาะสมกับขอบเขตของโครงงานที่เก็บข้อมูลด้วยไฟล์ JSON และมีการอัปโหลดสลิป/รูปภาพ

### ตัวอย่างการใช้งานในโครงงาน
- จัดเก็บ Source Code ของระบบ KruyMo บน GitHub ที่ https://github.com/PhisanuJay/KruyMo
- เผยแพร่เอกสารโครงงาน (ข้อเสนอ UML UAT รายงาน) ผ่านคลังโค้ดเดียวกัน
- สาธิตระบบจริงโดยรัน Backend ที่พอร์ต `3001` และ Frontend ที่พอร์ต `5173`

## สรุป

การประยุกต์ใช้เครื่องมือตามขั้นตอน SDLC ในโครงงาน KruyMo ช่วยให้ทีมสามารถวางแผน ออกแบบ พัฒนา ทดสอบ และเผยแพร่ระบบได้อย่างเป็นระบบ ตั้งแต่การกำหนดความต้องการ การจัดทำแผนภาพ UML การพัฒนาเว็บแอปพลิเคชันด้วย React และ Express การทดสอบด้วย UAT จนถึงการจัดเก็บโค้ดบน GitHub ซึ่งสอดคล้องกับแนวทางวิศวกรรมซอฟต์แวร์ของรายวิชา CSI 204
