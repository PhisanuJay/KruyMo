## 1. บทนำและขอบเขตการทดสอบ

การทดสอบ UAT นี้จัดทำขึ้นเพื่อยืนยันว่าระบบ KruyMo สามารถรองรับการทำงานตามความต้องการของผู้ใช้งานแต่ละกลุ่ม (Persona) ได้อย่างถูกต้องครบถ้วน ครอบคลุมกระบวนการหลักของธุรกิจตั้งแต่การจองชุดครุย การชำระเงิน การจัดส่งด้วยแมสเซนเจอร์ การรับคืนชุด และการคืนเงินมัดจำ

**ขอบเขต:** ทดสอบฟังก์ชันหลักของทั้ง 3 บทบาท คือ ลูกค้า (Customer), พนักงาน (Staff) และผู้ดูแลระบบ (Admin) รวมถึงการทดสอบกรณีผิดปกติ (Negative Test Case) และด้านความปลอดภัยของสิทธิ์การเข้าถึง

## 2. การวิเคราะห์ Persona ผู้ใช้งานหลัก

| Persona | คำอธิบาย | ความต้องการหลัก |
| --- | --- | --- |
| **Customer (ลูกค้า/บัณฑิต)** | นักศึกษาที่ต้องการเช่าชุดครุยสำหรับพิธีรับปริญญา | ค้นหา/จองชุด เลือกไซส์-ช่วงวันเช่า ชำระเงิน ติดตามสถานะ รับชุด และส่งคืนชุด |
| **Staff (พนักงาน)** | เจ้าหน้าที่ปฏิบัติการหน้าร้าน | ตรวจสอบสลิป อนุมัติการจอง เตรียมชุด จ่ายงานแมสเซนเจอร์ รับคืนชุดเข้าคลัง และคืนเงินมัดจำ |
| **Admin (ผู้ดูแลระบบ)** | ผู้จัดการระบบ/เจ้าของร้าน | จัดการชุดครุยและสต็อก จัดการข้อมูลหลัก จัดการผู้ใช้ ดูรายงาน/แดชบอร์ด และติดตามภาพรวมการดำเนินงาน |

## 3. การออกแบบ UAT Test Case (Test Case Design)

### 3.1 Persona: Customer

| รหัสทดสอบ | วัตถุประสงค์การทดสอบ | ข้อมูลนำเข้า (Input) | ขั้นตอนการทดสอบ | ผลลัพธ์ที่คาดหวัง (Expected Result) |
| --- | --- | --- | --- | --- |
| UAT-C01 | สมัครสมาชิก | ชื่อ, อีเมลใหม่, รหัสผ่าน, เบอร์โทร | ไปหน้า /register → กรอกข้อมูล → กดสมัคร | สร้างบัญชีสำเร็จ เข้าสู่ระบบอัตโนมัติในบทบาท customer |
| UAT-C02 | เข้าสู่ระบบ | customer@test.com / customer123 | ไปหน้า /login → กรอก → กดเข้าสู่ระบบ | เข้าสู่ระบบสำเร็จ กลับสู่หน้าลูกค้า แสดงชื่อผู้ใช้ |
| UAT-C03 | ลืมรหัสผ่าน | อีเมลที่มีในระบบ | ไปหน้า /forgot-password → กรอกอีเมล → กดส่ง | ระบบแจ้งว่าส่งลิงก์รีเซ็ตแล้ว |
| UAT-C04 | ค้นหา/กรองชุดครุยตามคณะ | เลือกคณะ / คำค้น | ไปหน้าแคตตาล็อก → เลือกคณะหรือพิมพ์คำค้น | แสดงเฉพาะชุดที่ตรงเงื่อนไข |
| UAT-C05 | ดูรายละเอียดชุด + เลือกไซส์/ระดับ/ช่วงวันเช่า | ไซส์, ระดับปริญญา, วันเริ่ม-วันคืน | เปิดหน้ารายละเอียดชุด → เลือกไซส์/ระดับ/วันที่ | คำนวณราคาเช่า+มัดจำตามจำนวนวันถูกต้อง ปุ่มจองพร้อมใช้งาน |
| UAT-C06 | เพิ่ม/ลบรายการโปรด | ชุดครุย 1 รายการ | กดหัวใจที่การ์ดชุด → เปิดหน้ารายการโปรด | ชุดถูกเพิ่ม/ลบจากรายการโปรดถูกต้อง |
| UAT-C07 | เพิ่มชุดลงตะกร้า | ชุด + ไซส์ + ช่วงวัน | เลือกเงื่อนไข → กดเพิ่มลงตะกร้า → เปิดตะกร้า | รายการปรากฏในตะกร้าพร้อมราค่าและช่วงวันเช่า |
| UAT-C08 | จองชุด + กรอกที่อยู่จัดส่ง | ที่อยู่ (ผู้รับ/เบอร์/บ้าน/แขวง/เขต/ไปรษณีย์) | กดจอง → หน้าเลื่อนขึ้นบนสุด → กรอกที่อยู่ → กดยืนยันการจอง | สร้างการจองสถานะ "รอชำระเงิน" และไปหน้าชำระเงิน |
| UAT-C09 | ชำระเงินด้วย PromptPay + อัปโหลดสลิป | รูปสลิปการโอน | สแกน QR / คัดลอกเบอร์พร้อมเพย์ → อัปโหลดสลิป → กดส่ง | สถานะเป็น "รอตรวจสอบ" แสดงข้อความรอดำเนินการ |
| UAT-C10 | ดูประวัติ/สถานะการจอง | — | เปิดเมนูการจองของฉัน → เลือกการจอง | แสดงรายการและไทม์ไลน์สถานะถูกต้อง |
| UAT-C11 | แก้ไขที่อยู่จัดส่งก่อนออกแมสฯ | ที่อยู่ใหม่ | เปิดรายละเอียดการจอง → แก้ไขที่อยู่ → บันทึก | ที่อยู่อัปเดตสำเร็จ (เฉพาะสถานะก่อนออกแมสฯ) |
| UAT-C12 | ยืนยันรับชุด (delivered) | — | สถานะ "กำลังจัดส่ง" → กดยืนยันได้รับชุด | สถานะเปลี่ยนเป็นได้รับชุด/กำลังใช้งาน |
| UAT-C13 | แจ้งส่งคืน + กรอกบัญชีรับเงินคืนมัดจำ | พร้อมเพย์/บัญชีธนาคาร + ชื่อบัญชี | กดแจ้งส่งคืน → เลือกช่องทางรับเงิน → กรอกบัญชี → ยืนยัน | บันทึกบัญชีรับเงินคืน สถานะเป็น "แจ้งส่งคืนแล้ว" |
| UAT-C14 | จัดการโปรไฟล์/เปลี่ยนรหัสผ่าน | ข้อมูลโปรไฟล์, รหัสผ่านใหม่ | เปิดหน้าโปรไฟล์ → แก้ไข/เปลี่ยนรหัส → บันทึก | ข้อมูลอัปเดต และเปลี่ยนรหัสผ่านสำเร็จ |
| UAT-C15 | การแจ้งเตือน (กระดิ่ง) | — | เมื่อสถานะการจองเปลี่ยน → เปิดกระดิ่งแจ้งเตือน | แสดงการแจ้งเตือนตามเหตุการณ์ล่าสุด |

### 3.2 Persona: Staff

| รหัสทดสอบ | วัตถุประสงค์การทดสอบ | ข้อมูลนำเข้า (Input) | ขั้นตอนการทดสอบ | ผลลัพธ์ที่คาดหวัง (Expected Result) |
| --- | --- | --- | --- | --- |
| UAT-S01 | เข้าสู่ระบบพนักงาน + แดชบอร์ด | staff@kruymo.com / staff123 | ล็อกอิน → เข้าหน้าแดชบอร์ดพนักงาน | เข้าสู่หน้า /staff แสดงคิวงานและสรุปสถานะ |
| UAT-S02 | ตรวจสอบสลิป/อนุมัติการชำระเงิน | การจองที่รอตรวจสลิป | เปิดคิวการจอง → เปิดสลิป → กดอนุมัติการชำระเงิน | สถานะเปลี่ยนเป็น "ชำระเงินแล้ว/อนุมัติ" |
| UAT-S03 | อนุมัติ/ปฏิเสธการจอง | เหตุผล (กรณีปฏิเสธ) | เลือกการจอง → กดอนุมัติ หรือปฏิเสธพร้อมเหตุผล | สถานะอัปเดตถูกต้อง แจ้งเตือนถึงลูกค้า |
| UAT-S04 | เตรียมชุด + จ่ายงานแมสเซนเจอร์ | ชื่อ/เบอร์แมสเซนเจอร์ | ติ๊ก checklist เตรียมชุด → กรอกข้อมูลแมสฯ → บันทึก | สถานะเป็น "พร้อมส่ง/รอแมสฯ" บันทึกข้อมูลแมสเซนเจอร์ |
| UAT-S05 | อัปเดตสถานะกำลังจัดส่ง | — | กดออกจัดส่ง (out_for_delivery) | สถานะเปลี่ยนเป็น "กำลังจัดส่ง" ลูกค้าเห็นสถานะ |
| UAT-S06 | รับคืนชุดเข้าคลัง | รูปสภาพชุด (ถ้ามี) | เลือกการจองที่ลูกค้าแจ้งส่งคืน → กดรับเข้าคลัง | สถานะเป็น "รับคืนแล้ว" พร้อมเข้าสู่ขั้นคืนมัดจำ |
| UAT-S07 | คืนเงินมัดจำ + หักค่าเสียหาย | ยอดหัก + เหตุผล (ถ้ามี) | เปิดหน้าคืนมัดจำ → ดูบัญชีลูกค้า → กำหนดยอดหัก → ยืนยันคืน | คำนวณยอดคืนสุทธิถูกต้อง สถานะเป็น "คืนมัดจำแล้ว" |

### 3.3 Persona: Admin

| รหัสทดสอบ | วัตถุประสงค์การทดสอบ | ข้อมูลนำเข้า (Input) | ขั้นตอนการทดสอบ | ผลลัพธ์ที่คาดหวัง (Expected Result) |
| --- | --- | --- | --- | --- |
| UAT-A01 | เข้าสู่ระบบแอดมิน + แดชบอร์ด | admin@kruymo.com / admin123 | ล็อกอิน → เข้าหน้าแดชบอร์ดผู้ดูแล | เข้าสู่ /admin แสดงคิวปฏิบัติการและตัวเลขสรุป |
| UAT-A02 | เพิ่มชุดครุย + จัดการสต็อก | ข้อมูลชุด + จำนวนสต็อกตามไซส์/ระดับ | หน้าจัดการชุด → เพิ่มชุดใหม่ → กำหนดสต็อก → บันทึก | ชุดใหม่และจำนวนสต็อกถูกบันทึกและแสดงในแคตตาล็อก |
| UAT-A03 | แก้ไข/ลบชุดครุย | ชุดที่มีอยู่ | เลือกชุด → แก้ไขข้อมูล/ลบ → บันทึก | ข้อมูลอัปเดต/ถูกลบถูกต้อง |
| UAT-A04 | จัดการข้อมูลหลัก (Master Data) | มหาวิทยาลัย/คณะ/ไซส์ | หน้าจัดการข้อมูลหลัก → เพิ่ม/แก้ไข → บันทึก | ข้อมูลหลักถูกบันทึกและนำไปใช้ในฟอร์มอื่น |
| UAT-A05 | ดูรายการจองทั้งหมด + รายละเอียด | — | หน้ารายการจองทั้งหมด → เปิดรายละเอียด 1 รายการ | แสดงรายการครบ พร้อมที่อยู่/แมสฯ/การคืน/บัญชีรับเงิน |
| UAT-A06 | จัดการผู้ใช้ (เพิ่ม/เปลี่ยนบทบาท) | ผู้ใช้ใหม่ + บทบาท | หน้าจัดการผู้ใช้ → เพิ่มผู้ใช้ / เปลี่ยนบทบาท → บันทึก | ผู้ใช้ถูกสร้าง/เปลี่ยนบทบาทสำเร็จ |
| UAT-A07 | ดูรายงานยอดขาย/สต็อก/ใกล้คืน | — | เปิดหน้ารายงาน | แสดงรายได้ สต็อกคงเหลือ และรายการใกล้ครบกำหนดคืนถูกต้อง |
| UAT-A08 | ติดตามคืนมัดจำ (view-only) | — | เปิดหน้าติดตามคืนมัดจำ | แสดงสรุปและรายการรอคืน/คืนแล้ว + บัญชีรับเงิน (ไม่มีปุ่มดำเนินการ) |
| UAT-A09 | ดูบันทึกกิจกรรม (Activity Log) | — | เปิดหน้า Activity Log | แสดงประวัติการทำรายการของผู้ใช้/พนักงาน |
| UAT-A10 | จัดการเทมเพลตการแจ้งเตือน | ข้อความเทมเพลต | เปิดหน้าเทมเพลตแจ้งเตือน → แก้ไข → บันทึก | เทมเพลตถูกบันทึกและใช้ส่งแจ้งเตือนตามสถานะ |

## 7. ตารางสรุปปัญหาที่พบ (Issue Report)

| Issue ID | รายละเอียดปัญหา | ประเภทปัญหา | ระดับความสำคัญ | อ้างอิง Test Case |
| --- | --- | --- | --- | --- |
| ISS-001 | ฟังก์ชัน "ลืมรหัสผ่าน" เป็นการจำลอง ไม่ได้ส่งอีเมลจริงและยังไม่มีหน้ารีเซ็ตรหัสผ่านจริง | ระบบทำงานไม่ตรงตามความต้องการ | Low | UAT-C03 |
| ISS-002 | ระบบยังไม่ป้องกันการส่ง/อัปโหลดสลิปการชำระเงินซ้ำ (ไม่มี idempotency) อาจเกิดรายการซ้ำ | ระบบทำงานไม่ตรงตามความต้องการ | Medium | UAT-N04 |

### แนวทางแก้ไข (Corrective Action)

- **ISS-002:** เพิ่มการตรวจสอบสถานะการชำระเงินก่อนรับสลิป และปิดปุ่ม/ล็อกการส่งซ้ำระหว่างประมวลผล
- **ISS-003:** วางแผนย้ายชั้นข้อมูลไปยังฐานข้อมูลจริง (เช่น PostgreSQL/MongoDB) พร้อมทำ transaction/locking ในกระบวนการจอง
